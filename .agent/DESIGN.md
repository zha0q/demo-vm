# Design.md：3D Mahjong Solitaire 视觉与交互规范

## 1. 设计目标

本游戏视觉参考所给首页、游戏页、通关页截图的整体气质，提炼为“东方禅意 + 木质器物 + 绿绒牌桌 + 金色奖励”的休闲解谜风格。

参考对象仅用于抽取设计语言，不复制原游戏的商标、Logo、素材、关卡截图、按钮图案、文案排版或受保护表达。

核心体验目标：

- 首页要像进入一扇东方木门，温暖、期待、轻松。
- 游戏页要像在绿色绒布桌上清牌，主体明确、UI 克制、按钮有质感。
- 通关页要有金色仪式感，突出“智慧、效率、奖励”。
- 所有页面优先适配竖屏移动端，整屏无滚动。

## 2. 视觉关键词

| 关键词 | 表现方式 |
|---|---|
| 东方 | 木门、竹影、窗棂、卷草纹、祥云、莲花、玉石感 |
| 休闲 | 圆润按钮、柔和阴影、低压反馈、暖色奖励 |
| 质感 | 木纹、绒布、金边、浮雕、牌面厚度 |
| 清晰 | 游戏主体居中，顶部信息简短，底部操作固定 |
| 奖励 | 通关页暗场聚光、金色标题、进度奖励条 |

| 游戏页 | 深玉绿、绿色绒布 | 木质按钮、奶油文字、绿色牌边 | 主体安静，牌面突出 |

- 不使用过多玻璃拟态。

## 4. 字体与文字

### 4.1 字体方向

优先使用系统可用中文字体，保证移动端清晰。

```css
font-family: "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif;
```

标题可使用更厚重的字重和金色描边效果。若后续引入字体资源，建议选择圆润、有游戏感、中文可读性强的标题字体。

### 4.2 字号建议

| 场景 | 字号 | 字重 | 颜色 |
|---|---:|---:|---|
| 首页大按钮文字 | 46-58px | 500-700 | `--color-cream-100` |
| 游戏页 IQ | 28-34px | 700 | `--color-gold-300` |
| 底部按钮副文本 | 20-24px | 700 | `--color-cream-100` |
| 通关页标题 | 58-76px | 700 | 金色渐变 |
| 通关统计标题 | 24-30px | 500 | `--color-gold-300` |
| 通关统计数值 | 38-48px | 700 | `--color-cream-100` |

### 4.3 描边与阴影

重要文字使用轻描边或多层阴影：

```css
text-shadow:
  0 2px 0 rgba(72, 28, 8, 0.75),
  0 4px 10px rgba(0, 0, 0, 0.35);
```

## 5. 布局基准

参考截图尺寸为 `864x1920`，按竖屏手机优先设计。

页面根容器：

```css
width: 100vw;
height: 100dvh;
overflow: hidden;
position: relative;
```

安全区：

```css
padding-top: max(24px, env(safe-area-inset-top));
padding-bottom: max(22px, env(safe-area-inset-bottom));
```

所有页面不允许出现浏览器滚动条。小屏高度不足时，优先压缩上下留白和 3D 相机取景，不压缩文字到不可读。

## 6. 首页规范

### 6.1 构图

首页是“木门入口”构图。

层级从后到前：

1. 暖色米纸窗背景。
2. 木质窗棂和中轴门缝。
3. 半透明竹影或叶片点缀。
4. 中上区域游戏标题。
5. 底部大号关卡按钮。
6. 左上头像按钮，右上音效和设置按钮。

### 6.2 背景

背景建议使用 CSS 多层渐变 + 纹理图片或程序化图案：

- 上半部为暖光木窗。
- 中轴有深木门缝。
- 底部为深木地台，带浅浮雕波纹。
- 左右可有轻微暗角。

如果暂无资产，可用渐变和伪元素实现第一版，不要使用纯色背景。

### 6.3 标题

标题位于屏幕上半部，约 `y = 300-580` 区间。

要求：

- 大号、厚重、奶油色或金白色。
- 深棕色厚描边。
- 可加入小麻将牌装饰，但不要使用参考图 Logo。
- 标题不能贴边，左右至少保留 `48px`。

### 6.4 顶部按钮

头像按钮：

- 位置：左上。
- 尺寸：约 `126x126px`。
- 形状：圆角方形，木框 + 米色头像剪影。

音效、设置按钮：

- 位置：右上。
- 尺寸：约 `72-82px` 圆形。
- 材质：木质圆按钮。
- 图标：奶油色。
- 阴影：下方深棕投影。

### 6.5 主按钮

首页主按钮位于底部偏上，参考截图约在屏幕高度 `78%-86%`。

样式：

- 胶囊形。
- 橙色渐变。
- 金色外边框。
- 深棕投影。
- 内部有高光反射。
- 文案如 `关卡 2`，字号大、居中。

尺寸建议：

```css
width: min(620px, calc(100vw - 160px));
height: 124px;
border-radius: 62px;
```

## 7. 游戏页规范

### 7.1 构图

游戏页是“绿绒桌面 + 中央牌局”构图。

从上到下：

1. 返回按钮、IQ、更多按钮。
2. 四槽步数队列。
3. 3D 麻将牌主体。
4. 底部三个操作按钮。

### 7.2 背景

背景使用深玉绿色绒布质感。

建议层级：

- 基础径向渐变：中心略亮，四角略暗。
- 细微布纹：低透明度噪声或线性纹理。
- 暗纹边框：四角可用低透明度花纹。
- 顶部可有轻微光照，避免纯黑。

颜色参考：

```css
background:
  radial-gradient(circle at 50% 34%, rgba(28, 133, 101, 0.45), transparent 42%),
  linear-gradient(145deg, #0d684e 0%, #07342e 62%, #031d1a 100%);
```

### 7.3 顶部栏

返回按钮：

- 左上圆形木质按钮。
- 尺寸：`72-82px`。
- 图标：奶油色左箭头。

更多按钮：

- 右上圆形木质按钮。
- 图标为三横线 + 点，奶油色。
- 展开后从右向左飘出静音、自动完成、重开按钮。

IQ：

- 顶部居中。
- 文案格式：`IQ: 105`。
- `IQ:` 可用橙金渐变，数字用浅奶油色。
- 初始为 `IQ: 0` 或按逻辑显示 `IQ: 100`，建议使用实际计算值。

### 7.4 步数队列

队列位于 IQ 下方，居中。

结构：

- 外框木质厚边。
- 内部深棕底。
- 四个等宽槽位。
- 槽位之间有细分隔线。
- 队列整体圆角较大，但不要胶囊过度。

尺寸建议：

```css
width: min(410px, calc(100vw - 220px));
height: 92px;
border-radius: 16px;
```

视觉状态：

| 状态 | 表现 |
|---|---|
| 空槽 | 深棕底，低对比 |
| 有牌 | 小麻将牌居中，轻阴影 |
| 匹配 | 两牌靠近、金色闪光 |
| 队列满 | 队列轻微横向抖动 |

### 7.5 麻将牌

牌面：

- 奶油白或象牙白。
- 牌角圆润。
- 右下和侧边有绿色厚边。
- 正面图案使用红、蓝、绿三色。
- 图案风格要清晰、粗线、适合小屏。

3D 质感：

- 牌厚度明显。
- 层叠时有阴影。
- 可选牌边缘更亮，绿色描边或微光。
- 不可选牌略暗，但不要看不清。

参考尺寸比例：

```txt
tile width : tile height ≈ 1 : 1.25
tile depth : tile width ≈ 0.12-0.18
```

### 7.6 游戏主体区域

主体牌局位于屏幕中部，视觉宽度约占屏幕 `88%-94%`。

要求：

- 牌堆整体居中。
- 顶部队列和底部按钮不遮挡牌。
- 层级最高的牌更亮，底层牌有阴影但仍可识别。
- 点击命中区域以牌面为准，不因阴影扩大误触。

### 7.7 底部按钮

底部按钮三枚，横向分布：

- 洗牌或关卡选择。
- 提示。
- 撤回。

样式：

- 圆形深棕底。
- 木质外环。
- 奶油色图标。
- 按钮下方可显示 `Lv. 6` 或功能短文本。
- 右上红点显示剩余次数。
- 锁定状态显示小锁圆点。

尺寸建议：

```css
.action-button {
  width: 118px;
  height: 118px;
  border-radius: 999px;
}
```

红点：

```css
.badge {
  min-width: 42px;
  height: 42px;
  border-radius: 21px;
  background: linear-gradient(#e94221, #a91308);
  color: #fff8df;
}
```

## 8. 通关页规范

### 8.1 构图

通关页是在游戏页上叠加暗场奖励弹层。

层级：

1. 原游戏页背景和按钮被暗化。
2. 中央金色光晕。
3. 顶部莲花或东方纹样。
4. 大标题。
5. 三个统计卡片。
6. 鼓励文案。
7. 奖励进度条。
8. 底部绿色下一关按钮。

### 8.2 背景遮罩

使用黑绿遮罩，顶部和底部更暗，中间有金色聚光。

```css
background:
  radial-gradient(circle at 50% 32%, rgba(238, 161, 37, 0.35), transparent 26%),
  rgba(0, 10, 8, 0.86);
```

原游戏 UI 可隐约可见，但不能抢主内容。

### 8.3 标题

标题如 `智慧超群`，居中。

样式：

- 金色渐变。
- 大字号。
- 深色描边或阴影。
- 上方可有莲花金色图案。
- 左右可有祥云装饰，但不要过密。

### 8.4 统计卡片

三张并排小卡：

- 时间。
- IQ。
- 连击。

样式：

- 半透明深棕面板。
- 金色描边。
- 上方标题小，底部数值大。
- 图标放在数值左侧。

尺寸建议：

```css
width: 224px;
height: 148px;
border-radius: 10px;
```

在窄屏中按比例缩小整体间距，卡片仍保持三列，不换行。

### 8.5 鼓励文案

文案位于统计卡片下方。

风格：

- 奶油金文字。
- 关键数字使用亮绿色。
- 两侧或上下有细分隔线。

示例文案方向：

```txt
极简打法！仅 1.97 个托牌，令人印象深刻！
```

注意：具体文案可变，但不要直接照抄参考截图用于上线。

### 8.6 奖励进度条

结构：

- 一排小格子。
- 当前进度格为橙色亮起。
- 末端为宝箱或奖励图标。
- 下方显示目标，如 `达到第10关`。

### 8.7 主按钮

通关页主按钮为绿色胶囊按钮。

样式：

- 绿色渐变。
- 深绿色底部阴影。
- 内部高光。
- 文案大号居中，如 `关卡 2` 或 `下一关`。

尺寸建议：

```css
width: min(550px, calc(100vw - 220px));
height: 116px;
border-radius: 58px;
```

## 9. 图标规范

图标应统一为“厚实、圆角、填充型”。

| 图标 | 方向 |
|---|---|
| 返回 | 粗左箭头 |
| 更多 | 三横线 + 圆点 |
| 设置 | 齿轮 |
| 音效 | 音符或扬声器 |
| 洗牌 | 双箭头交叉 |
| 提示 | 灯泡 |
| 撤回 | 回转箭头 |
| 锁定 | 小锁 |
| 时间 | 时钟 |
| IQ | 星章或脑力标识 |
| 连击 | 靶心或连击环 |

图标颜色优先使用 `--color-cream-100` 或 `--color-gold-300`。

## 10. 动效规范

### 10.1 首页

- 标题进入：轻微下落 + 弹性缩放。
- 叶片：慢速飘动，可循环。
- 主按钮：轻微呼吸高光。
- 木门背景不做大幅运动。

### 10.2 游戏页

- 点击可选牌：牌轻抬、边缘亮起。
- 点击不可选牌：牌或按钮轻微抖动。
- 入队：牌从牌桌飞向队列槽。
- 匹配：两张队列牌移动到相邻位置并碰撞。
- 消除：碎裂成块并向下坠落，随后淡出。
- 提示：目标牌绿色或金色脉冲 `1.2s`。
- 撤回：牌从队列或消除位置回到牌桌。

### 10.3 通关页

- 暗场淡入：`180-240ms`。
- 金色光晕扩散：`320-480ms`。
- 标题弹入：轻微缩放。
- 统计卡片：从下向上依次出现。
- 主按钮：延迟出现并带柔和高光。

## 11. 组件状态

### 11.1 按钮状态

| 状态 | 表现 |
|---|---|
| 默认 | 木质/绿色/橙色渐变，清晰阴影 |
| 按下 | 向下移动 2-4px，阴影缩短 |
| 禁用 | 降低亮度与饱和度，显示锁或灰化 |
| 免费 | 右上可显示绿色 `Free` 小标 |
| 有次数 | 右上红点显示数字 |

### 11.2 牌状态

| 状态 | 表现 |
|---|---|
| 可选 | 正常亮度 + 轻微绿色边缘光 |
| 不可选 | 亮度降低 12%-18%，无边缘光 |
| 悬停 | 轻抬或加亮 |
| 已选 | 金色描边或上浮 |
| 队列中 | 缩小显示在槽位中 |
| 匹配中 | 发光、靠近、碰撞 |
| 消除中 | 碎裂、坠落、淡出 |

## 12. 适配规范

### 12.1 竖屏优先

主设计尺寸：`864x1920`。

兼容尺寸：

- `375x667`
- `390x844`
- `430x932`
- `768x1024`

### 12.2 固定与自适应

固定：

- 顶部按钮尺寸。
- IQ 字号。
- 队列槽位比例。
- 底部按钮尺寸。
- 牌面基础比例。

自适应：

- 顶部到队列间距。
- 队列到牌桌间距。
- 牌桌相机缩放。
- 牌桌到底部按钮间距。
- 首页标题与按钮的垂直位置。

### 12.3 禁止项

- 页面不能滚动。
- 底部按钮不能遮挡牌桌。
- 队列不能超出屏幕。
- 标题和按钮文字不能溢出容器。
- 通关页统计卡片不能换行到两行。

## 14. 与当前工程的落地建议

当前项目已有：

- `src/styles.less`
- `src/render/mahjongScene.ts`
- `src/App.tsx`
- `.agent/PLAN.md`

推荐落地顺序：

1. 在 `src/styles.less` 建立本文件的 CSS 变量。
2. 先把页面背景从当前 demo 风格改为首页、游戏页、通关页三套主题。

4. 更新 `MahjongScene` 的背景色、牌材质、可选高亮和阴影。
5. 添加队列槽位、底部圆形按钮、红点和通关暗场。
6. 用浏览器检查四个目标尺寸，确认无滚动和无遮挡。
