# 麻将连连看规则

## 核心原则

把游戏规则保持为纯函数、确定性逻辑，并且独立于 React 和 Three.js。渲染层可以询问一张牌是否可选，但不能决定规则真相。

## 规则清单

- 只有在牌处于激活状态、没有被更高层牌覆盖、且至少有一个水平侧面开放时，牌才可被选择。
- 更高层覆盖判断要看 x/y 投影是否重叠，以及 `z` 是否更高。
- 左右阻挡判断要检查同层中与牌垂直范围重叠的邻居。
- 普通牌只与相同牌面匹配。
- 花牌 `F1` 到 `F4` 互相匹配。
- 季节牌 `S1` 到 `S4` 互相匹配。
- 已移除或处于动画中的牌都不能被选择。
- 队列容量由 `STEP_QUEUE_MAX_SIZE` 决定，当前为 4。
- 队列匹配时，应优先让新加入的牌去匹配最早入队的那张可匹配牌。

## 测试驱动要求

在实现之前，先为要修改的确切规则添加或更新 Vitest 覆盖。

优先使用聚焦测试：

- `test/matchRules.test.ts`：牌面分组和匹配。
- `test/boardRules.test.ts`：可选/开放牌逻辑。
- `test/stepQueue.test.ts`：队列容量和立即匹配行为。
- `test/hintService.test.ts`：提示选择优先级。
- `test/shuffleService.test.ts`：洗牌不变式。
- `test/solvabilityChecker.test.ts`：可清空/死局位置。

运行：

```bash
npm test
```

## 常见错误

| 错误 | 修正 |
|---|---|
| 只检查直接相邻，忽略覆盖 | 用投影重叠加更高 `z` 判断覆盖 |
| 让 Three.js 网格边界决定规则 | 用 `src/game` 中的规则坐标 |
| 把花牌或季节牌当成完全相同的牌面 | 按分组匹配 |
| 在辅助函数里悄悄改状态 | 优先使用纯函数，或清晰命名的命令函数 |
| 洗位置而不是洗牌面 | 只洗牌面，保持布局固定 |
