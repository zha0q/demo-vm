# 3D 麻将场景（Three.js）

## 核心原则

Three.js 负责渲染游戏状态并回传用户意图。它不能掌管规则状态，不能决定匹配结果，也不能修改分数或绕过 `src/game` 里的辅助函数。

## 场景职责

- 根据 `GameState` 渲染激活牌。
- 高亮可选、悬停、选中、提示、队列中、匹配中和动画中的状态。
- 对点击进行射线拾取，并把 `tileId` 回传给 React/游戏状态。
- 把激活棋盘框在可用视口内。
- 动画运行期间锁定或忽略输入。
- 在卸载时释放几何体、材质、事件监听器和动画帧。

## 坐标映射

规则空间使用 `x`、`y`、`z`，其中 `z` 表示层高。Three.js 中 `y` 表示垂直高度。

```ts
worldX = tile.x * horizontalUnit;
worldY = tile.z * layerRise;
worldZ = tile.y * verticalUnit;
```

所有规则判断都要放在 `src/game` 中。不要用网格包围盒来决定一张牌是否开放。

## 动画建议

- 选中：轻微抬升或发光，不改变布局。
- 进入队列：牌从棋盘位置飞向队列槽位。
- 匹配：两张队列牌靠近、碰撞，然后结算。
- 移除：简单碎片或块状网格落下并淡出。
- 洗牌：位置固定时做翻转、闪光或换面动画。
- 撤销：先视觉上把牌恢复到棋盘，再提交恢复后的状态。

## 浏览器验证

在修改场景或布局后：

```bash
npm run build
npm run dev
```

在以下尺寸下验证：

- 375x667
- 390x844
- 430x932
- 768x1024

检查：

- 画布不是空白。
- 牌面取景正常且可点击。
- 有效开放牌会出现高亮。
- UI 不会覆盖棋盘。
- 不会出现浏览器滚动条。

## 常见错误

| 错误 | 修正 |
|---|---|
| 每一帧都重建所有网格 | 尽量按 tile id 复用网格 |
| 卸载时遗留事件监听器 | 在 `dispose()` 中移除所有监听器 |
| 让摄像机在移动端漂走 | 在关卡变化和窗口缩放后重新取景 |
| 把动画状态混进规则辅助函数 | 把临时动画状态留在渲染/UI 层 |
