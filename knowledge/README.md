# 知识库

这个目录保存麻将连连看游戏的持久项目知识。只要实现决策、架构边界、玩法规则、设计令牌、测试策略或 Agent 工作流约定发生变化，就应该更新这里的文档。

## 使用方式

修改相关区域前，请先阅读这些文件：

| 文件 | 使用场景 |
|---|---|
| `architecture.md` | 修改模块边界、应用状态、React 流程、Three.js 集成、持久化 |
| `game-rules.md` | 修改麻将规则、匹配、步骤队列、提示、撤销、洗牌、可解性、计分、IQ |
| `design-system.md` | 修改页面视觉、布局、按钮、牌面、颜色、动效、移动端适配 |
| `testing-and-qa.md` | 添加测试、修改 QA 流程、浏览器验证、构建命令 |
| `agent-maintenance.md` | 收尾 Agent 工作、更新知识、更新技能、维护任务文档 |
| `decision-log.md` | 记录持久决策和理由 |

## 维护规则

每次 Agent 开发任务结束时：

- 如果任务改变了项目行为、架构、设计、测试或工作流，请更新相关的 `knowledge/*.md` 文档。
- 当重复工作流、常见坑或项目特定规则应该指导未来 Agent 时，请更新 `.agent/skills/*/SKILL.md`。
- 对不显而易见的产品或工程决策，在 `decision-log.md` 添加带日期的记录。
- 不要重复粘贴完整实现细节；改为链接或点名对应源码文件。

## 事实来源优先级

当文档之间不一致时，按以下顺序解决：

1. 当前源码和测试。
2. `.agent/PLAN.md` 中的预期实现计划。
3. `.agent/Design.md` 中的视觉方向。
4. `knowledge/*.md` 中的持久摘要和理由。
5. `.agent/skills/*/SKILL.md` 中的 Agent 流程。

如果发现不匹配，请在同一个任务里更新过期文档。
