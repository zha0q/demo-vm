# 决策日志

在这里记录持久决策。条目保持简短并带日期。

## 2026-06-20

### 使用项目级 `.agent` 文档和技能

决策：

- 将实现计划放在 `.agent/PLAN.md`。
- 将视觉设计规范放在 `.agent/Design.md`。
- 将可复用的 Agent 工作流放在 `.agent/skills`。
- 将持久项目知识放在 `knowledge/`。

理由：

- 这个项目既需要实现指导，也需要紧凑且经常更新的知识。
- `.agent/skills` 用来指导未来 Agent 的行为。
- `knowledge/` 用来保存应当跨任务保留的稳定事实和理由。

### 把 IQ 视为本局鼓励分

决策：

- IQ 只代表本局。
- 它使用进度、连击和耗时。
- 不应该被呈现为真实智力测量。

理由：

- 这样既能保留激励作用，也不会夸大其现实意义。

### 保持规则逻辑与 Three.js 分离

决策：

- 规则真相位于 `src/game`。
- Three.js 只负责渲染和回传点击。

理由：

- 纯规则更容易测试。
- 渲染和动画可以演进，而不会污染游戏状态。

### 先内联页面组件，再后续拆分 `src/ui`

决策：

- 首页、游戏页和完成页已先在 `src/App.tsx` 内落地。
- `src/game` 规则模块优先拆分，UI 组件后续再移动到 `src/ui`。

理由：

- 当前目标是按计划快速形成可玩的单屏流程。
- 规则模块更需要稳定测试边界；UI 拆文件可以在行为稳定后做，不影响用户流程。
