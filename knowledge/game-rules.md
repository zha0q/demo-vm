# 游戏规则知识

## 游戏类型

本游戏是一个 3D 堆叠式麻将连连看配对游戏，带四格步骤队列。它借鉴的是麻将连连看这一大类玩法模式，不使用受版权保护的关卡布局或素材。

## 牌的可选条件

只有同时满足以下条件时，牌才可以被选择：

- 牌存在。
- 牌处于激活状态，且未被移除。
- 没有更高层的激活牌在 x/y 投影上覆盖它。
- 同一层中至少有一个水平侧面是开放的。

可选性属于规则层概念。不要使用 Three.js 网格边界来做规则判断。

## 配对匹配

普通牌：

- 只能与相同牌面匹配。

花牌：

- `F1`、`F2`、`F3`、`F4` 可以互相匹配。

季节牌：

- `S1`、`S2`、`S3`、`S4` 可以互相匹配。

花牌不能与季节牌匹配。

## 步骤队列

游戏使用顶部步骤队列：

- 最大容量为 `4`。
- 被选中的开放牌进入队列。
- 如果新入队的牌与已有队列牌匹配，该对子立即移除。
- 如果多张队列牌都能匹配，选择最早入队的匹配牌。
- 如果队列已满，且新牌不能立即匹配，则拒绝本次移动。
- MVP 不应该仅因为队列已满就触发失败。
- 当前实现由 `src/game/stepQueue.ts` 负责纯队列规则，`src/App.tsx` 在点击开放牌时接入队列并调用 `removePair` 完成消除。

视觉行为：

- 当前 UI 已显示四格托牌槽。
- 飞入、碰撞、破碎/掉落仍是后续 Three.js 动画任务。

## 提示

提示优先级：

1. 用开放的棋盘牌匹配已有队列牌。
2. 匹配两张开放的棋盘牌。
3. 如果没有可用提示，显示不可用反馈。

MVP 建议：如果没有提示，不消耗提示次数。

## 洗牌

洗牌应该：

- 保持牌的位置和层级不变。
- 在剩余激活牌之间重新分配牌面。
- 保持可配对性。
- 至少产生一个可用移动。
- 可以被撤销。
- 可选地要求洗牌后仍可解。

洗牌不应该改变已移除的牌。

## 撤销

撤销应该恢复所有影响游戏的状态：

- 牌。
- 队列。
- 分数。
- 移动次数。
- 连击和最大连击。
- 剩余行动次数。
- IQ 输入。
- 如有必要，也恢复运行时消息。

为可靠性，使用快照命令。

## 可解性

可解性检查器应该：

- 枚举当前可选的匹配对子。
- 模拟移除对子。
- 递归搜索，直到清台或死局。
- 对状态哈希做记忆化。
- 遵守节点/时间限制。

运行时可解性检查应该避免阻塞 UI。

## 计分

计分与 IQ 分离。

初始建议分数：

```ts
baseMatchScore = 100;
comboBonus = currentCombo * 15;
timeBonus = Math.max(0, Math.round(30 - elapsedSeconds / 10));
score += baseMatchScore + comboBonus + timeBonus;
```

通过测试和玩法手感继续调整计分。

## 局内 IQ

IQ 只代表本局表现。不能把它呈现为真实智力测量。

输入：

- `clearedPairs`
- `totalPairs`
- `currentCombo`
- `elapsedSeconds`

权重：

- 进度：45%
- 时间效率：30%
- 连击：25%

边界：

- 最小值：85
- 最大值：155

开局置信度保护：

- 在进度达到 12% 前，IQ 更贴近 100，避免早期波动过大。

标签：

- 85-99：稳定观察中
- 100-114：思路在线
- 115-129：节奏不错
- 130-144：高效清台
- 145-155：神级状态

当前实现：

- `src/game/inGameIqCalculator.ts` 使用计划公式并处理 `totalPairs <= 0`，避免 `NaN`。
- 游戏页展示“清台 IQ”和标签，完成页展示最终 IQ 与标签。
- IQ 不用于存档排名或跨关卡比较。
